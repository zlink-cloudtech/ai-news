## 抓取方式（关键发现）

机器之心**没有公开 RSS**。官方 `/rss` 端点 2026-06 实测返回 HTML 营销页。
实际数据源是其 SPA 站点的 **GraphQL endpoint**。

### 必要的请求头
1. `X-CSRF-Token`: 从首页 `<meta name="csrf-token">` 提取
2. `X-Requested-With: XMLHttpRequest` （**关键**！否则返回 HTML 而非 JSON）
3. `Origin: https://www.jiqizhixin.com`
4. `Referer: https://www.jiqizhixin.com/`

### GraphQL 查询
1. **列表**（一次拿 30 条，按 published_at 倒序）:
   ```graphql
   {
     articles(first: 30) {
       edges {
         node {
           id
           title
           path
           published_at
           description
           cover_image_url
           category_name
           author { name }
         }
         cursor
       }
       pageInfo { hasNextPage endCursor }
     }
   }
   ```
2. **详情**（用 `node()` 全局查询，兼容 ID 中的斜杠）:
   ```graphql
   {
     node(id: "Article/<uuid>") {
       id
       ... on Article {
         title
         path
         published_at
         description
         simple_content
         content
         foreword
       }
     }
   }
   ```

### 字段说明
- `id` 格式：`Article/<uuid>`（在 `article(id:)` 中带斜杠会报 parse error；用 `node(id:)`）
- `published_at` 格式：`"2026/06/13 11:19"`（不带时区，按 UTC+8 处理）
- `description` / `foreword` / `simple_content` 经常为空；`content` 是完整 HTML 正文
- `path` 是相对路径（如 `/articles/2026-06-13-5`），需 `urljoin` 成完整 URL
- `category_name` 是中文分类（"产业" / "工程" / "学界" / "产品" / "投资" 等）

## 抓取流程
1. 拉首页 → 拿 CSRF token
2. GraphQL `articles(first: 30)` 拿列表
3. 解析 `published_at` → Python datetime
4. 按窗口过滤
5. 对窗口内每篇调 `node(id:)` 拿 `content`（HTML），清洗成纯文本作 summary
6. 串行抓详情，默认 delay=1.0s

## 抓取示例
```bash
# 不抓全文（快）
python3 scripts/crawlers/crawl_jiqizhixin.py --days 1 --no-article
# 抓全文（慢，每篇 1s）
python3 scripts/crawlers/crawl_jiqizhixin.py --date 2026-06-13 --delay 1.0
```

## 备注
- 30 条容量限制 + 列表只看首页前 30，跨多日运行时会丢数据
  → 当前实现：每次只过滤 `resolve_dates(args)` 给出的日期窗口
  → 改进方向：可加 cursor 分页拉更多（如 100 条）覆盖一周窗口
- 内容质量很高：产业/工程/学术分类清晰，作者署名（机器之心/方恩瑞等）
- 多数文章有 `content` 字段（约 2000-5000 字符），可作为下游 LLM 总结的优质输入
- 反爬：1s delay 暂未触发限流；如被封可加 Random jitter / Referer 旋转
