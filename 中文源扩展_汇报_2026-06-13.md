## 4. 遇到的问题和解决方案

### 问题 1：36kr RSS 时间格式非标准
**症状**：`published` 字段是 `"2026-06-13 16:37:21  +0800"`（双空格 + 4位时区），`parsedate_to_datetime` 直接抛 `Invalid date value or format`。结果 `_utils.parse_any_datetime` 返回 `None`，所有条目 `published_ts=None`，**全部被窗口过滤掉**。

**解决**：在 `_utils.py:parse_rss` 加兜底逻辑 —— 如果 `parse_any_datetime` 返回 None 但 feedparser 拿到了 `published_parsed`，就用 `calendar.timegm(struct_time)` 按 UTC 解释（feedparser 已经把时区转好了）。

**额外修了一个 bug**：第一版 36kr 关键词正则有 trailing `|`，导致 `re.search` 永远匹配空串，把"比亚迪泰国"这种非 AI 文章也放进来。重构成 `"|".join(_KW_PATTERNS)` 列表写法。

### 问题 2：机器之心无公开 RSS
**症状**：
- `https://www.jiqizhixin.com/rss` → 200 但返回 HTML 营销页
- `https://www.jiqizhixin.com/feed` → 404
- `https://www.jiqizhixin.com/feed/atom` → 404
- WordPress 风格路径（`/articles/feed`、`/feed.xml` 等）全部 404 或返回 HTML
- 首页是 SPA，HTML 中无任何文章 URL

**解决**：
1. 翻 JS bundle（`application-9d263b208c0be6cf8f5f.js`）找到 `graphql-tag` 引用 + `/graphql` 端点字符串
2. 试 GraphQL 直连 → 200 但返回 HTML。**关键发现**：必须带 `X-Requested-With: XMLHttpRequest` 头才返回 JSON
3. 进一步试 `article(id: ...)` → null。**关键发现**：用 `node(id: ...)` + `... on Article { ... }` 内联片段才能拿到详情
4. 还需要 `X-CSRF-Token`（从首页 `<meta name="csrf-token">` 提取）才能正常 POST
5. 跑通完整的列表+详情流程：`articles(first: 30)` → 窗口过滤 → 对窗口内每篇调 `node(id:)` 拿 `content` HTML → 清洗成纯文本作 summary

### 问题 3：机器之心 detail 的 summary 字段组合
**症状**：第一次实现时 `description + simple_content + foreword` 拼接出空 summary（机器之心的 `description`/`foreword`/`simple_content` 经常为空，只有 `content` 字段有完整 HTML）

**解决**：改为从 4 个候选字段（description / foreword / simple_content / content）中选**最长非空**那个，清洗 HTML 后截 800 字符作 summary。

### 问题 4：沙箱依赖缺失
**症状**：第一次跑时报 `ModuleNotFoundError: No module named 'feedparser'`

**解决**：`pip3 install feedparser trafilatura cloudscraper beautifulsoup4 lxml`（cloudscraper 早已装，其他几个补装）
