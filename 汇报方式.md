# 汇报方式

> AI 资讯追踪系统的资讯展示和推送方式说明。

## 当前阶段：GitHub 直接看

- **存储位置**：`https://github.com/zlink-cloudtech/ai-news`
- **每日资讯路径**：`每日资讯/YYYY-MM-DD.md`
- **每周汇总路径**：`每周汇总/YYYY-WXX.md`
- **查看方式**：浏览器访问 GitHub 仓库
- **优势**：
  - 零额外配置，git 仓库本身就是产品
  - 提交即推送，自动化链路最短
  - 可以 review 历史每日资讯
  - 支持 Markdown 表格/链接/标签 完整渲染

## 触发时点

| 任务 | 触发时间 | 抓取范围 | 推送方式 |
|------|---------|---------|---------|
| 每日资讯 | 每天 8:30（CST） | 昨日 00:00 - 23:59 | 自动 commit + push（post-commit hook） |
| 每周汇总 | 每周日早 8:30（CST） | 本周一 - 周日 | 自动 commit + push |

## 后续升级路径

| 阶段 | 方式 | 触发条件 |
|------|------|---------|
| **当前** | GitHub 直接看 | 已就位 |
| **第一阶段升级** | 飞书 Webhook 推送摘要卡片 | 每日资讯格式确认稳定（运行 1-2 周） |
| **第二阶段升级** | 邮件订阅日报 | 用户主动要求 |
| **第三阶段升级** | 实时增量推送（12:00/18:00/22:00） | 需要小时级时效时启用 |

## 升级到飞书 Webhook 时的技术要求

参考调研报告，升级到飞书推送需要：

- 飞书机器人创建 + Webhook URL
- 每日资讯生成后调用 Webhook API
- 卡片格式设计（标题 / 摘要 / 关键 3 条 / 查看完整日报 按钮）
- 通知频率限制：≤ 100 次/日（飞书免费版限流）

## 不做的方式（已确认）

- ❌ 邮件订阅：信息密度低、查看不便
- ❌ 微信公众号推送：受限于订阅号每日 1 条限制
- ❌ 即时通讯群机器人：噪音大、干扰日常工作
- ❌ 实时推送：当前阶段无小时级时效需求
